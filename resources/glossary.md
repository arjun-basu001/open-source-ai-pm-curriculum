# Glossary

A quick-reference glossary for terms used across this curriculum.

---

**Agent** — A system that uses an LLM to decide and execute actions (calling tools, running code, reading/writing data) to complete a goal.

**Chain-of-thought (CoT)** — A reasoning style where the model works through intermediate steps before giving a final answer.

**Context window** — The total token budget a model can process in one call (input + output).

**Embedding** — A vector representation of data (usually text) that preserves semantic similarity.

**Evaluation (evals)** — A repeatable method for measuring output quality with test cases and scoring rules.

**Fine-tuning** — Additional training on a task-specific dataset to alter model behavior.

**Foundation model** — A large general model that can be adapted to many downstream tasks.

**Function calling / Tool use** — Structured model outputs that trigger external tool/API execution.

**Grounding** — Anchoring generation in trusted external evidence (often via retrieval).

**Hallucination** — Confident but incorrect output generated by a model.

**Inference** — Running a trained model to produce outputs.

**LLM-as-judge** — Using one model to score or assess outputs from another model.

**Latency** — Time from request submission to response receipt.

**Prompt injection** — Malicious/hostile input intended to override instructions or leak data.

**RAG (Retrieval-Augmented Generation)** — Retrieval + generation architecture that adds relevant external context at inference time.

**RLHF** — Reinforcement Learning from Human Feedback; a method for aligning model behavior to human preferences.

**System prompt** — High-priority instructions set at the beginning of a model interaction.

**Temperature** — Sampling control for output randomness; lower = more deterministic, higher = more diverse.

**Token** — The unit models process text in; cost, latency, and context are all token-based.

**Vector database** — Data store optimized for similarity search over embeddings.
