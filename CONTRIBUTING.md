# Contributing

Thanks for wanting to improve this curriculum — seriously.

This project is opinionated on purpose: we care about practical depth, not link dumping. If you open a PR, aim to make the learner’s next hour better.

## What great contributions look like

- A clearly better resource (more current, more practical, or easier to apply)
- A new exercise that ends in a **real artifact** someone can review
- Domain-specific versions (healthcare, fintech, legal, education, etc.)
- Broken link fixes and factual updates
- Translations that preserve the original clarity and tone

## What we usually won’t accept

- "Tool of the week" additions with no lasting value
- Sponsored, affiliate, or promotional links
- Generic rewrites that water down the point of a module
- Placeholder sections or unfinished content

## How to contribute

1. Fork the repo
2. Create a focused branch: `git checkout -b improve-<topic>`
3. Make your changes (small scope beats giant PRs)
4. Open a PR with:
   - what changed,
   - why it helps learners,
   - and sources for any new factual claims

## Writing style guide

Write like a seasoned practitioner helping a teammate:

- Be direct
- Be concrete
- Name trade-offs
- Skip hype
- Skip fluff

If you’re unsure whether something belongs, open an issue first and we’ll decide together.
